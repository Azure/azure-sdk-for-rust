# Using the Event Hubs SDK from a modern C++ application

`azure_messaging_eventhubs` is a pure-Rust crate. It exposes a Rust async API
and ships no C ABI, so a C++ program cannot link against it directly. To call it
from C++ you build a thin Rust "shim" crate that re-exports the functionality you
need through `extern "C"` functions, generate a C header for that shim, and wrap
the resulting flat C API in RAII C++ types.

This document walks through that full path end to end:

## 1. Why a shim is required

Three properties of the Rust API make a direct C++ binding impossible and shape
everything in the shim:

- **No stable ABI.** Rust structs, enums, `Result`, `String`, and `Vec` have no
  guaranteed memory layout. Only `#[repr(C)]` types and `extern "C"` functions are
  callable from C/C++. The shim is the layer that translates between the rich Rust
  types and a flat C surface.
- **Everything is `async`.** Every I/O method (`open`, `send_event`, `send_batch`,
  `stream_events`, `close`) returns a future and must be driven by an async
  runtime. C++ has no awareness of Rust futures, so the shim owns a Tokio runtime
  and exposes *blocking* C functions that call `runtime.block_on(...)` internally.
- **Trait objects and generics don't cross FFI.** Credentials are passed as
  `Arc<dyn TokenCredential>`, sends are generic over `impl Into<EventData>`, and
  the receiver returns `impl Stream`. None of these can appear in a C signature,
  so the shim picks one concrete behavior (for example, `DeveloperToolsCredential`
  and a byte-slice body) and hides the generics behind opaque handles.

The boundary contract is intentionally narrow: pointers, integers, lengths, and
`#[repr(C)]` structs. Richer constructs (futures, trait objects, ownership
semantics) stay on the Rust side and are mediated through functions. This
"blocking adapter over an async core" is the standard pattern for exposing a Rust
async library to a synchronous host language: the runtime lives in the shim, not
the caller. Opaque handles, pointers the caller never dereferences, keep
non-FFI-safe Rust types such as `ProducerClient` and `EventReceiver` alive across
calls without ever describing their layout to C++.

## 2. Architecture

```
+------------------------+
|  C++ application        |   RAII types: Producer, Consumer, Receiver, Event
|  (namespace azure::     |   std::string_view in, std::span<const std::byte> out,
|   eventhubs)            |   exceptions on error
+-----------+------------+
            | links against, calls flat C functions
            v
+------------------------+
|  eventhubs.h            |   generated by cbindgen: extern "C" decls,
|  (generated C header)   |   #[repr(C)] structs, opaque handle typedefs
+-----------+------------+
            | C ABI (eh_producer_open, eh_producer_send_event, ...)
            v
+------------------------+
|  azure_messaging_       |   #[no_mangle] extern "C" fns, opaque handles,
|  eventhubs_ffi          |   owns a tokio::Runtime, block_on(...) per call,
|  (Rust cdylib/staticlib)|   catch_unwind, thread-local last-error
+-----------+------------+
            | normal Rust calls
            v
+------------------------+
|  azure_messaging_       |   ProducerClient / ConsumerClient / EventReceiver
|  eventhubs (Rust crate) |   async API, Arc<dyn TokenCredential>
+------------------------+
```

The two artifacts you author are the **shim crate** (Rust) and the **C++ wrapper**
(header-only is fine). The C header in the middle is generated.

## 3. The Rust FFI shim crate

Create a sibling crate, for example `sdk/eventhubs/azure_messaging_eventhubs_ffi`.

### 3.1 Cargo.toml

```toml
[package]

version = "0.1.0"
edition = "2021"
publish = false

[lib]
# cdylib produces a shared library (.so/.dylib/.dll); staticlib produces a .a/.lib.
# Build both so the C++ side can link either way.

[dependencies]
azure_messaging_eventhubs = { path = "../azure_messaging_eventhubs" }
azure_identity = { workspace = true }
azure_core = { workspace = true }
futures = { workspace = true }
# A multi-thread runtime: the SDK spawns background tasks for its AMQP links,
# so a current-thread runtime is not sufficient.

[build-dependencies]

```

### 3.2 The runtime, handles, and error channel

```rust
// src/lib.rs
//
// A flat C ABI over azure_messaging_eventhubs. Every exported function is
// extern "C", panic-safe, and reports failure through an i32 status code plus a
// thread-local last-error string.

use std::cell::RefCell;
use std::ffi::{c_char, c_int, CStr, CString};
use std::panic::{catch_unwind, AssertUnwindSafe};
use std::ptr;
use std::sync::{Arc, OnceLock};

use azure_core::credentials::TokenCredential;
use azure_identity::DeveloperToolsCredential;
use azure_messaging_eventhubs::{
    ConsumerClient, EventReceiver, ProducerClient, ReceivedEventData,
};
use futures::StreamExt;
use tokio::runtime::Runtime;

// One process-wide runtime. The SDK clients hold AMQP connections that depend on
// the runtime staying alive, so it must outlive every handle: a static is the
// simplest way to guarantee that.
fn runtime() -> &'static Runtime {
    static RT: OnceLock<Runtime> = OnceLock::new();
    RT.get_or_init(|| {
        Runtime::new().expect("failed to build Tokio runtime for the Event Hubs FFI shim")
    })
}

// Status codes returned across the boundary. 0 is success.
pub const EH_OK: c_int = 0;
pub const EH_ERR_INVALID_ARG: c_int = 1;
pub const EH_ERR_UTF8: c_int = 2;
pub const EH_ERR_SDK: c_int = 3;
pub const EH_ERR_PANIC: c_int = 4;
pub const EH_ERR_END_OF_STREAM: c_int = 5;

thread_local! {
    static LAST_ERROR: RefCell<Option<CString>> = const { RefCell::new(None) };
}

fn set_last_error(message: impl Into<String>) {
    let msg = message.into();
    // Replace interior NULs so the string is always a valid C string.
    let cstr = CString::new(msg.replace('\0', " ")).unwrap_or_default();
    LAST_ERROR.with(|slot| *slot.borrow_mut() = Some(cstr));
}

/// Returns the last error message recorded on the calling thread, or NULL if
/// there is none. The pointer is owned by the shim and stays valid until the
/// next failing call on the same thread. Callers must copy it if they need it
/// longer; they must not free it.
#[no_mangle]
pub extern "C" fn eh_last_error() -> *const c_char {
    LAST_ERROR.with(|slot| {
        slot.borrow()
            .as_ref()
            .map(|s| s.as_ptr())
            .unwrap_or(ptr::null())
    })
}

// Helper: turn a C string pointer into a &str, recording an error on failure.
fn cstr_to_str<'a>(ptr: *const c_char) -> Result<&'a str, c_int> {
    if ptr.is_null() {
        set_last_error("unexpected null pointer argument");
        return Err(EH_ERR_INVALID_ARG);
    }
    // Safety: the caller contract requires a NUL-terminated UTF-8 C string.
    unsafe { CStr::from_ptr(ptr) }.to_str().map_err(|_| {
        set_last_error("argument was not valid UTF-8");
        EH_ERR_UTF8
    })
}

// Helper: run the body, converting panics into a status code so an unwind never
// crosses the FFI boundary (which would be undefined behavior).
fn guard(f: impl FnOnce() -> c_int) -> c_int {
    match catch_unwind(AssertUnwindSafe(f)) {
        Ok(code) => code,
        Err(_) => {
            set_last_error("a panic was caught at the FFI boundary");
            EH_ERR_PANIC
        }
    }
}

fn credential() -> Result<Arc<dyn TokenCredential>, c_int> {
    DeveloperToolsCredential::new(None)
        .map(|c| c as Arc<dyn TokenCredential>)
        .map_err(|e| {
            set_last_error(format!("failed to construct credential: {e}"));
            EH_ERR_SDK
        })
}
```

Three choices in this layer carry the design. `OnceLock<Runtime>` provides a
lazily-initialized, never-dropped runtime, which is appropriate for a
process-wide service client and avoids the "runtime dropped while a connection is
still open" class of bugs. The thread-local last-error channel mirrors `errno`
and `GetLastError`: each function returns a small status code, and a separate
accessor yields the human-readable detail; keeping the string thread-local avoids
data races when multiple C++ threads call the shim concurrently. The
`catch_unwind` guard on every exported function is mandatory rather than optional,
because a Rust panic unwinding into C++ frames is undefined behavior; converting
it to a status code is what keeps the boundary sound.

### 3.3 Producer functions

```rust
// Opaque to C: the header sees only a forward-declared struct pointer.
pub struct EhProducer {
    client: ProducerClient,
}

/// Opens a producer for `namespace`/`eventhub` using DeveloperToolsCredential.
/// On success writes a non-null handle to `out` and returns EH_OK. The caller
/// owns the handle and must release it with eh_producer_close.
#[no_mangle]
pub extern "C" fn eh_producer_open(
    namespace: *const c_char,
    eventhub: *const c_char,
    out: *mut *mut EhProducer,
) -> c_int {
    guard(|| {
        if out.is_null() {
            set_last_error("out parameter was null");
            return EH_ERR_INVALID_ARG;
        }
        let namespace = match cstr_to_str(namespace) {
            Ok(s) => s,
            Err(code) => return code,
        };
        let eventhub = match cstr_to_str(eventhub) {
            Ok(s) => s,
            Err(code) => return code,
        };
        let credential = match credential() {
            Ok(c) => c,
            Err(code) => return code,
        };

        let result = runtime().block_on(async {
            ProducerClient::builder()
                .open(namespace, eventhub, credential)
                .await
        });

        match result {
            Ok(client) => {
                let handle = Box::new(EhProducer { client });
                // Transfer ownership to the caller as a raw pointer.
                unsafe { *out = Box::into_raw(handle) };
                EH_OK
            }
            Err(e) => {
                set_last_error(format!("producer open failed: {e}"));
                EH_ERR_SDK
            }
        }
    })
}

/// Sends a single event whose body is the `len` bytes at `body`. The byte slice
/// is copied before the call returns, so the caller may free it immediately.
#[no_mangle]
pub extern "C" fn eh_producer_send_event(
    producer: *mut EhProducer,
    body: *const u8,
    len: usize,
    partition_id: *const c_char, // may be NULL for automatic partition assignment
) -> c_int {
    guard(|| {
        let Some(producer) = (unsafe { producer.as_ref() }) else {
            set_last_error("producer handle was null");
            return EH_ERR_INVALID_ARG;
        };
        if body.is_null() && len != 0 {
            set_last_error("body was null but len was non-zero");
            return EH_ERR_INVALID_ARG;
        }
        // Safety: caller guarantees `body` points to `len` readable bytes.
        let bytes = if len == 0 {
            Vec::new()
        } else {
            unsafe { std::slice::from_raw_parts(body, len) }.to_vec()
        };

        let options = if partition_id.is_null() {
            None
        } else {
            match cstr_to_str(partition_id) {
                Ok(p) => Some(azure_messaging_eventhubs::SendEventOptions {
                    partition_id: Some(p.to_string()),
                }),
                Err(code) => return code,
            }
        };

        let result = runtime()
            .block_on(async { producer.client.send_event(bytes, options).await });

        match result {
            Ok(()) => EH_OK,
            Err(e) => {
                set_last_error(format!("send_event failed: {e}"));
                EH_ERR_SDK
            }
        }
    })
}

/// Closes the producer and frees the handle. Passing NULL is a no-op.
#[no_mangle]
pub extern "C" fn eh_producer_close(producer: *mut EhProducer) -> c_int {
    guard(|| {
        if producer.is_null() {
            return EH_OK;
        }
        // Reclaim ownership of the Box and run the async close before dropping.
        let handle = unsafe { Box::from_raw(producer) };
        let result = runtime().block_on(async { handle.client.close().await });
        match result {
            Ok(()) => EH_OK,
            Err(e) => {
                set_last_error(format!("producer close failed: {e}"));
                EH_ERR_SDK
            }
        }
    })
}
```

Note how `send_event` relies on the crate's `impl<T: Into<Vec<u8>>> From<T> for
EventData`: passing a `Vec<u8>` is enough, the SDK wraps it as `EventData`
internally. That conversion is what lets the C side think purely in terms of
bytes and lengths.

### 3.4 Consumer and receiver functions

The receiver is the subtle part. `EventReceiver::stream_events(&self)` returns
`impl Stream<...> + '_`: a stream that *borrows* the receiver. To pull one event
per C call you must keep the stream alive between calls, which means storing the
receiver and a stream that borrows it in the same handle. That is a
self-referential struct, which Rust's borrow checker forbids by default.

The shim resolves this by boxing the receiver (giving it a stable heap address)
and extending the borrowed stream's lifetime to `'static` with a tightly-scoped
`unsafe`. The drop order in the struct guarantees the stream is destroyed before
the receiver it borrowed.

```rust
type EventStream =
    std::pin::Pin<Box<dyn futures::Stream<Item = azure_messaging_eventhubs::Result<ReceivedEventData>> + Send>>;

pub struct EhConsumer {
    client: ConsumerClient,
}

pub struct EhReceiver {
    // Field order matters: `stream` is dropped before `receiver`, so the
    // borrow the stream holds is always released first.
    stream: EventStream,
    receiver: Box<EventReceiver>,
}

pub struct EhEvent {
    body: Vec<u8>,
    sequence_number: i64,
}

#[no_mangle]
pub extern "C" fn eh_consumer_open(
    namespace: *const c_char,
    eventhub: *const c_char,
    out: *mut *mut EhConsumer,
) -> c_int {
    guard(|| {
        if out.is_null() {
            set_last_error("out parameter was null");
            return EH_ERR_INVALID_ARG;
        }
        let namespace = match cstr_to_str(namespace) {
            Ok(s) => s,
            Err(code) => return code,
        };
        let eventhub = match cstr_to_str(eventhub) {
            Ok(s) => s.to_string(),
            Err(code) => return code,
        };
        let credential = match credential() {
            Ok(c) => c,
            Err(code) => return code,
        };

        let result = runtime().block_on(async {
            ConsumerClient::builder()
                .open(namespace, eventhub, credential)
                .await
        });

        match result {
            Ok(client) => {
                unsafe { *out = Box::into_raw(Box::new(EhConsumer { client })) };
                EH_OK
            }
            Err(e) => {
                set_last_error(format!("consumer open failed: {e}"));
                EH_ERR_SDK
            }
        }
    })
}

/// Opens a receiver on a partition, starting from the earliest available event.
#[no_mangle]
pub extern "C" fn eh_consumer_open_receiver(
    consumer: *mut EhConsumer,
    partition_id: *const c_char,
    out: *mut *mut EhReceiver,
) -> c_int {
    guard(|| {
        let Some(consumer) = (unsafe { consumer.as_ref() }) else {
            set_last_error("consumer handle was null");
            return EH_ERR_INVALID_ARG;
        };
        if out.is_null() {
            set_last_error("out parameter was null");
            return EH_ERR_INVALID_ARG;
        }
        let partition_id = match cstr_to_str(partition_id) {
            Ok(s) => s.to_string(),
            Err(code) => return code,
        };

        let options = Some(azure_messaging_eventhubs::OpenReceiverOptions {
            start_position: Some(azure_messaging_eventhubs::StartPosition {
                location: azure_messaging_eventhubs::StartLocation::Earliest,
                ..Default::default()
            }),
            ..Default::default()
        });

        let receiver = match runtime().block_on(async {
            consumer
                .client
                .open_receiver_on_partition(partition_id, options)
                .await
        }) {
            Ok(r) => Box::new(r),
            Err(e) => {
                set_last_error(format!("open_receiver_on_partition failed: {e}"));
                return EH_ERR_SDK;
            }
        };

        // Build a stream that borrows the boxed receiver, then extend its
        // lifetime to 'static.
        //
        // Safety: `receiver` lives on the heap at a stable address inside the
        // same EhReceiver we are about to return, and `stream` is declared
        // before `receiver` so it is dropped first. The stream therefore never
        // outlives the receiver it borrows.
        let stream: EventStream = {
            let receiver_ref: &EventReceiver = &receiver;
            let static_ref: &'static EventReceiver =
                unsafe { std::mem::transmute(receiver_ref) };
            Box::pin(static_ref.stream_events())
        };

        unsafe { *out = Box::into_raw(Box::new(EhReceiver { stream, receiver })) };
        EH_OK
    })
}

/// Blocks until the next event arrives, writing a freshly-allocated event handle
/// to `out`. Returns EH_ERR_END_OF_STREAM if the stream has finished.
#[no_mangle]
pub extern "C" fn eh_receiver_next(
    receiver: *mut EhReceiver,
    out: *mut *mut EhEvent,
) -> c_int {
    guard(|| {
        let Some(receiver) = (unsafe { receiver.as_mut() }) else {
            set_last_error("receiver handle was null");
            return EH_ERR_INVALID_ARG;
        };
        if out.is_null() {
            set_last_error("out parameter was null");
            return EH_ERR_INVALID_ARG;
        }

        match runtime().block_on(async { receiver.stream.next().await }) {
            Some(Ok(event)) => {
                let body = event.event_data().body().unwrap_or_default().to_vec();
                let handle = Box::new(EhEvent {
                    body,
                    sequence_number: event.sequence_number().unwrap_or_default(),
                });
                unsafe { *out = Box::into_raw(handle) };
                EH_OK
            }
            Some(Err(e)) => {
                set_last_error(format!("receive failed: {e}"));
                EH_ERR_SDK
            }
            None => EH_ERR_END_OF_STREAM,
        }
    })
}

/// Borrows the event body. The returned pointer is valid until eh_event_free is
/// called on this event; the caller must not free it.
#[no_mangle]
pub extern "C" fn eh_event_body(
    event: *const EhEvent,
    out_ptr: *mut *const u8,
    out_len: *mut usize,
) -> c_int {
    guard(|| {
        let Some(event) = (unsafe { event.as_ref() }) else {
            set_last_error("event handle was null");
            return EH_ERR_INVALID_ARG;
        };
        if out_ptr.is_null() || out_len.is_null() {
            set_last_error("out parameter was null");
            return EH_ERR_INVALID_ARG;
        }
        unsafe {
            *out_ptr = event.body.as_ptr();
            *out_len = event.body.len();
        }
        EH_OK
    })
}

#[no_mangle]
pub extern "C" fn eh_event_sequence_number(event: *const EhEvent) -> i64 {
    match unsafe { event.as_ref() } {
        Some(e) => e.sequence_number,
        None => -1,
    }
}

#[no_mangle]
pub extern "C" fn eh_event_free(event: *mut EhEvent) {
    if !event.is_null() {
        drop(unsafe { Box::from_raw(event) });
    }
}

#[no_mangle]
pub extern "C" fn eh_receiver_close(receiver: *mut EhReceiver) -> c_int {
    guard(|| {
        if receiver.is_null() {
            return EH_OK;
        }
        let handle = unsafe { Box::from_raw(receiver) };
        // Drop the stream first (releases the borrow), then close the receiver.
        let EhReceiver { stream, receiver } = *handle;
        drop(stream);
        let receiver = *receiver;
        match runtime().block_on(async { receiver.close().await }) {
            Ok(()) => EH_OK,
            Err(e) => {
                set_last_error(format!("receiver close failed: {e}"));
                EH_ERR_SDK
            }
        }
    })
}

#[no_mangle]
pub extern "C" fn eh_consumer_close(consumer: *mut EhConsumer) -> c_int {
    guard(|| {
        if consumer.is_null() {
            return EH_OK;
        }
        let handle = unsafe { Box::from_raw(consumer) };
        match runtime().block_on(async { handle.client.close().await }) {
            Ok(()) => EH_OK,
            Err(e) => {
                set_last_error(format!("consumer close failed: {e}"));
                EH_ERR_SDK
            }
        }
    })
}
```

The self-referential receiver is the only place this shim requires hand-written
`unsafe`. Its soundness rests on field drop order: struct fields are dropped top
to bottom, so declaring `stream` before `receiver` guarantees the borrow is
released before the value it borrows is destroyed. The `ouroboros` and
`self_cell` crates encode the same pattern without hand-written `unsafe` if that
is preferred. Ownership across the boundary is explicit and one-directional:
`Box::into_raw` transfers a heap object to C++, and `Box::from_raw` in the
matching `*_close`/`*_free` reclaims it; every `into_raw` must have exactly one
`from_raw`, or the object is leaked or double-freed. `eh_event_body` returns a
borrow into the event rather than a copy, which keeps the receive path
allocation-free on the C++ side in exchange for the documented rule that the
pointer is valid only until the event is freed.

## 4. Generating the C header

Drive cbindgen from a `build.rs` so the header regenerates on every build.

```rust
// build.rs
fn main() {
    let crate_dir = std::env::var("CARGO_MANIFEST_DIR").unwrap();
    cbindgen::Builder::new()
        .with_crate(&crate_dir)
        .with_language(cbindgen::Language::C)
        .with_pragma_once(true)
        .with_documentation(true)
        .generate()
        .expect("cbindgen failed to generate the C header")
        .write_to_file("include/eventhubs.h");
}
```

The generated `include/eventhubs.h` declares the opaque handle types and the
functions. It looks roughly like this:

```c
#pragma once

```

## 5. The modern C++ wrapper

The flat C API is correct but unpleasant to use directly: manual close calls,
status-code checks, raw pointers. A header-only C++ layer turns it into idiomatic
modern C++: RAII handles that close themselves, exceptions instead of status
codes, and `std::string_view`/`std::span` at the edges.

```cpp
// eventhubs.hpp - a header-only modern C++ wrapper (C++20).
#pragma once

#include <cstddef>
#include <cstdint>
#include <memory>
#include <optional>
#include <span>
#include <stdexcept>
#include <string>
#include <string_view>

extern "C" {
#include "eventhubs.h"
}

namespace azure::eventhubs {

// Thrown for any non-OK status from the C layer. Carries the code and the
// thread-local detail string captured at the failure site.
class Error : public std::runtime_error {
public:
    Error(int code, std::string what)
        : std::runtime_error(std::move(what)), code_(code) {}
    int code() const noexcept { return code_; }

private:
    int code_;
};

namespace detail {
// Translate a status code into either a return or an exception.
inline void check(int status, const char* op) {
    if (status == EH_OK) return;
    const char* detail = eh_last_error();
    std::string message = std::string(op) + " failed (code " +
                          std::to_string(status) + ")";
    if (detail != nullptr) {
        message += ": ";
        message += detail;
    }
    throw Error(status, std::move(message));
}
} // namespace detail

// RAII view over a received event. Frees the underlying handle on destruction.
class Event {
public:
    explicit Event(EhEvent* handle) noexcept : handle_(handle) {}
    Event(Event&&) noexcept = default;
    Event& operator=(Event&&) noexcept = default;
    Event(const Event&) = delete;
    Event& operator=(const Event&) = delete;

    // The returned span borrows memory owned by this Event; it is valid only
    // for the lifetime of the Event.
    std::span<const std::byte> body() const {
        const uint8_t* ptr = nullptr;
        size_t len = 0;
        detail::check(eh_event_body(handle_.get(), &ptr, &len), "eh_event_body");
        return {reinterpret_cast<const std::byte*>(ptr), len};
    }

    std::string_view body_as_text() const {
        auto bytes = body();
        return {reinterpret_cast<const char*>(bytes.data()), bytes.size()};
    }

    int64_t sequence_number() const noexcept {
        return eh_event_sequence_number(handle_.get());
    }

private:
    struct Deleter {
        void operator()(EhEvent* p) const noexcept { eh_event_free(p); }
    };
    std::unique_ptr<EhEvent, Deleter> handle_;
};

class Receiver {
public:
    explicit Receiver(EhReceiver* handle) noexcept : handle_(handle) {}

    // Blocks for the next event. Returns std::nullopt at end of stream.
    std::optional<Event> next() {
        EhEvent* raw = nullptr;
        int status = eh_receiver_next(handle_.get(), &raw);
        if (status == EH_ERR_END_OF_STREAM) return std::nullopt;
        detail::check(status, "eh_receiver_next");
        return Event{raw};
    }

private:
    struct Deleter {
        void operator()(EhReceiver* p) const noexcept { eh_receiver_close(p); }
    };
    std::unique_ptr<EhReceiver, Deleter> handle_;
};

class Consumer {
public:
    static Consumer open(std::string_view namespace_, std::string_view eventhub) {
        EhConsumer* raw = nullptr;
        detail::check(
            eh_consumer_open(std::string(namespace_).c_str(),
                             std::string(eventhub).c_str(), &raw),
            "eh_consumer_open");
        return Consumer{raw};
    }

    Receiver open_receiver(std::string_view partition_id) {
        EhReceiver* raw = nullptr;
        detail::check(
            eh_consumer_open_receiver(handle_.get(),
                                      std::string(partition_id).c_str(), &raw),
            "eh_consumer_open_receiver");
        return Receiver{raw};
    }

private:
    explicit Consumer(EhConsumer* handle) noexcept : handle_(handle) {}
    struct Deleter {
        void operator()(EhConsumer* p) const noexcept { eh_consumer_close(p); }
    };
    std::unique_ptr<EhConsumer, Deleter> handle_;
};

class Producer {
public:
    static Producer open(std::string_view namespace_, std::string_view eventhub) {
        EhProducer* raw = nullptr;
        detail::check(
            eh_producer_open(std::string(namespace_).c_str(),
                             std::string(eventhub).c_str(), &raw),
            "eh_producer_open");
        return Producer{raw};
    }

    // partition_id is optional: pass std::nullopt for automatic assignment.
    void send(std::span<const std::byte> body,
              std::optional<std::string_view> partition_id = std::nullopt) {
        std::string partition;
        const char* partition_ptr = nullptr;
        if (partition_id) {
            partition = std::string(*partition_id);
            partition_ptr = partition.c_str();
        }
        detail::check(
            eh_producer_send_event(
                handle_.get(),
                reinterpret_cast<const uint8_t*>(body.data()), body.size(),
                partition_ptr),
            "eh_producer_send_event");
    }

    void send(std::string_view text,
              std::optional<std::string_view> partition_id = std::nullopt) {
        send(std::as_bytes(std::span{text.data(), text.size()}), partition_id);
    }

private:
    explicit Producer(EhProducer* handle) noexcept : handle_(handle) {}
    struct Deleter {
        void operator()(EhProducer* p) const noexcept { eh_producer_close(p); }
    };
    std::unique_ptr<EhProducer, Deleter> handle_;
};

} // namespace azure::eventhubs
```

The C++ layer applies three standard idioms. `std::unique_ptr` with a stateless
custom deleter binds each C handle to C++ object lifetime, so the matching
`*_close`/`*_free` runs exactly once, including during exception unwinding.
`Event::body()` returns a `std::span` that borrows the event's storage, expressing
the C ABI's lifetime rule ("valid only while the Event lives") in the type system
rather than in a comment. Converting status codes to exceptions at the boundary
lets application code read as ordinary C++, with the detail from `eh_last_error()`
carried in the exception message.

## 6. A complete C++ program

```cpp
// main.cpp
#include "eventhubs.hpp"

#include <cstdlib>
#include <iostream>

namespace eh = azure::eventhubs;

int main() {
    const char* ns = std::getenv("EVENTHUBS_HOST");        // e.g. myns.servicebus.windows.net
    const char* hub = std::getenv("EVENTHUB_NAME");
    if (ns == nullptr || hub == nullptr) {
        std::cerr << "set EVENTHUBS_HOST and EVENTHUB_NAME\n";
        return 1;
    }

    try {
        // Produce a few events.
        auto producer = eh::Producer::open(ns, hub);
        for (int i = 0; i < 3; ++i) {
            producer.send("hello from C++ #" + std::to_string(i));
        }
        std::cout << "sent 3 events\n";

        // Consume from partition 0.
        auto consumer = eh::Consumer::open(ns, hub);
        auto receiver = consumer.open_receiver("0");
        for (int received = 0; received < 3; ++received) {
            auto event = receiver.next();
            if (!event) break; // end of stream
            std::cout << "seq=" << event->sequence_number()
                      << " body=" << event->body_as_text() << "\n";
        }
        // producer, consumer, receiver all close in reverse construction order
        // as they leave scope.
    } catch (const eh::Error& e) {
        std::cerr << "Event Hubs error: " << e.what() << "\n";
        return 1;
    }
    return 0;
}
```

Authentication uses `DeveloperToolsCredential`, which picks up an Azure CLI
login, so run `az login` before launching the program. To use a different
credential (managed identity in production, for instance), change the single
`credential()` function in the shim; the C and C++ layers do not need to know.

## 8. Safety, threading, and lifetime rules

These are the contracts the C++ side must honor. They are the price of crossing
the FFI boundary, and most "works in Rust, crashes from C++" bugs trace back to
violating one of them.

- **One open, one close.** Every handle returned through an `out` parameter is
  owned by the caller and must be released exactly once with its matching
  `*_close`/`*_free`. The RAII wrappers in section 5 guarantee this; if you call
  the C API directly, you are on the hook. The wrappers also forbid copying
  (only move), so a handle cannot be double-freed.
- **Borrows die with their owner.** `eh_event_body` returns a pointer into the
  event; it is invalid after `eh_event_free`. The `std::span` from
  `Event::body()` carries the same rule. Copy the bytes if you need them to
  outlive the event.
- **Blocking calls block the calling thread.** `eh_producer_send_event`,
  `eh_receiver_next`, and the `open`/`close` calls run async work to completion on
  the shim's Tokio runtime, parking the calling C++ thread until they return.
  `eh_receiver_next` in particular blocks until an event arrives or the receive
  times out. Call it from a worker thread if your application has a UI or event
  loop to keep responsive.
- **Handles are not thread-safe by default.** Do not call methods on the *same*
  handle from two threads at once. Distinct handles on distinct threads are fine;
  the runtime and the thread-local error channel are built for that. If you need
  to share one producer across threads, serialize access with a mutex on the C++
  side.
- **Panics never escape.** Every exported function is wrapped in `catch_unwind`
  and converts a panic into `EH_ERR_PANIC`, so a Rust-side bug surfaces as an
  exception in C++ rather than undefined behavior. It still indicates a bug worth
  reporting.
- **UTF-8 in, UTF-8 out.** All string arguments must be NUL-terminated UTF-8.
  Namespaces, hub names, and partition IDs are ASCII in practice, but event
  bodies are arbitrary bytes and are handled as byte slices, not strings, for
  exactly that reason.

### Extending the shim

The surface above covers single-event produce and partition-receive. The same
patterns extend to the rest of the crate:

- **Batched sends:** wrap `ProducerClient::create_batch` /
  `EventDataBatch::try_add_event_data` / `send_batch` behind an `EhBatch` opaque
  handle. `try_add_event_data` returns `false` when the batch is full, which maps
  naturally to a C function returning a "would not fit" status.
- **Event metadata:** `ReceivedEventData` exposes `offset()`, `partition_key()`,
  `enqueued_time()`, and `system_properties()`. Add `eh_event_*` accessors that
  copy each field into the `EhEvent` handle the way `sequence_number` already does.
- **Properties and message IDs:** `EventData::builder()` supports
  `with_content_type`, `with_message_id`, `with_correlation_id`, and
  `add_property`. Expose a builder-style set of C functions if your application
  needs to set them, rather than the single byte-body shortcut used here.

Each addition follows the same three steps: add the `extern "C"` function in the
shim, regenerate the header, and add a method to the corresponding C++ RAII type.
